The EverLost Project

[ HyperLink to the Everlost page ]

Everlost was the second game I ever made. It was where I gained my most valuable experiences. Unfortunately, the project was never completed due to a lack of a clear end goal. I kept adding things, and the project just grew bigger. The difficulty of working on the project never increased because I always considered future scenarios.

But I will never forget this project as it is a reminder that hard things never come easy, so just keep going.

EverLost featured multiplayer support, controller support, keybinding, and local multiplayer. The game is about a hiker who gets forever lost in the mountains, even if you retrace your steps, it would be completely new terrain, slay monsters and stay alive for as long as you can. (Made In GameMaker Studios 2)

Streamed Holistic Tracking

I used a ESP32-CAM from a robot I was building at the time to stream video capture over wifi (since BTS protocals are to weak), the board its self also hosted a Wireless Access Point so my phone could connect to it and see the video stream.

I then had a Python server on my PC that connected to the stream over wifi, the program would then sample video frames from the stream to use as input for mediapipe's holistic tracking model which would display via OpenCV.

Project used >> MediaPipe, ESP32-CAM, OpenCV, C++, Python

This Is WinterSpark

This is a ambitious project, my over all goal is to make a realistic dragon mask of WinterSpark that would mimic my facial features. I also wanted to use the mask for exploring so I intergated night vision into it.

My Working Plan

I am using a ESP32-CAM (the AI thinker board model) that is connected directly to the command line (TX1, RX0) of a Arduino Nano (a small microcontroller board) to stream video capture by jpeg through the USB serial protocol to a Raspberry pi (model 4).

The Raspberry pi will be moded to run a script when its powered on, that script then connects to the Arduino nano's COM port and grabs the current video frame, after filtering it through of couple of filters and Cascades to find the pupils of the eyes, the program will find the right angle and adjust both servo motors accordingly (X axis servo, Y axis servo) on the eyes of the mask, these servos will be connected and controlled by the raspberry pi.

Finally I have 2 extra micro cameras that run on 15 fps / 60 mW, these are directly connected to small LCD screens in the mask thanks to the help of 2 multiplexers (they expand 4 (digital or analog) pins from 4 to 16 outputs), all of this is also connected to the Arduino nano but this circuit does not live on the same level as the eye tracking circuit, in other words its a separate process but on the same board, these circuits don't know that each other exists.

Project used >> OpenCV, Ardunio Nano, ESP32-CAM, Rasberry Pi, C++, Python
